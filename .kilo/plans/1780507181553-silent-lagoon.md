# QACopilot Complete Redesign Plan

## ⚠️ Hard Constraint: Zero Breaking Changes to Existing Functionality

**No redesign work may break, remove, or alter any existing working feature.** Every enhancement must be additive or purely cosmetic. All current user flows must continue to work exactly as before:
- AI test case generation (all providers)
- Test script generation (all frameworks/languages)
- Knowledge Base upload, search, chunking
- Auth (register, login, forgot-password)
- API key management (encrypted storage)
- Excel & PDF export
- History (restore, delete, clear)
- Supabase & local storage modes

Implementation must follow **incremental, parallel-track migration** — old code stays functional until new code is proven equivalent.

---

## 🎯 Objective

Full redesign and enhancement of QACopilot — a QA Test Case & Script Generation platform — focusing on modern UI/UX, expanded functionality, improved architecture, and scalability — all without disrupting existing working features.

---

✅ AI-powered test case generation (RAG-based)

❌ No collaboration features
❌ Weak mobile experience
❌ No orchestration / CI hooks
❌ Server uses raw Node.js handlers instead of a framework

## 🚀 Phase 1: Foundation & Architecture (Weeks 1–3)

### 1.1 Codebase Restructuring
- **Migrate monolith to feature-based folder structure**
  ```
  src/
    features/
      auth/
      generator/
      test-scripts/
      knowledge/
      settings/
      history/
    shared/
      components/
      hooks/
      utils/
      types/
    app/
  ```
- **Introduce state management** — Zustand or Redux Toolkit for global state (auth, settings, history)
- **Add React Query (TanStack Query)** for server state caching, background refetch, optimistic updates
- **Standardize API client** — extend `axios` instance with typed hooks

### 1.2 Design System
- Create shared component library in `src/shared/components/ui/`:
  - `Button`, `Input`, `Select`, `Textarea`, `Modal`, `Toast`, `Table`, `Tabs`, `Dropdown`, `Badge`, `Avatar`, `Tooltip`
- Define **design tokens** via Tailwind CSS theme config:
  - Colors (primary, surface, border, text scales)
  - Spacing, typography, radius, shadows
  - Dark mode palette
- Enforce component usage — ban raw Tailwind in page components for common primitives

### 1.3 Routing & Navigation
- Replace manual `activePage` state with **React Router v7** (file-based routing)
- Add route guards for auth
- Breadcrumb navigation
- URL-based deep linking for history items

### 1.4 Backend Improvements
- Migrate raw Node.js handler to **Express.js** (or Fastify) for middleware ecosystem
  - OR keep lightweight but extract route handlers into separate files
- Add **Zod** for request validation
- Standardize error response format
- Add request logging (pino / morgan)
- Add rate limiting per route

## 🎨 Phase 2: UI/UX Overhaul (Weeks 4–6)

### 2.1 Dark Mode
- Full dark mode support via CSS variables / Tailwind `dark:` classes
- Persist user preference (localStorage + user profile)
- Smooth theme transition

### 2.2 Dashboard (New Page)
- **Overview cards**: total test cases generated, knowledge files, API usage, recent activity
- **Charts** (via Recharts):
  - Test cases by category (positive/negative/validation/edge)
  - Generation trend over time
  - Provider usage distribution
- Quick actions from dashboard

### 2.3 Generator Page Redesign
- Split-pane layout: Input left, results right
- **Inline editing** of generated test cases:
  - Click-to-edit fields (summary, description, steps, expected)
  - Add/remove/reorder test steps
  - Add/edit/delete test cases manually
  - Clone test case
- Bulk actions: select multiple → delete, export selected only
- Advanced filtering & sorting by category, TC_ID
- Collapsible RAG context section with preview
- Template library for common requirements (Login, Signup, Checkout, Search…)

### 2.4 Test Scripts Page Redesign
- Framework selector with card UI + descriptions
- Script preview with syntax highlighting (Monaco or Shiki)
- Copy to clipboard, download as file
- Language/framework combination matrix with badges
- Script customization panel (browser, headless toggle, env vars)

### 2.5 Knowledge Base Redesign
- Grid/List view toggle for files
- Drag-and-drop upload zone (already has react-dropzone — enhance UI)
- File preview panel (PDF thumbnail, DOCX preview, image OCR preview)
- Chunk viewer with search/filter
- Upload progress indicators
- Bulk delete, bulk re-chunk

### 2.6 Global UX Enhancements
- Skeleton loaders instead of spinners
- Toast notification system (success/error/info)
- Confirmation dialogs for destructive actions (already has one — standardize)
- Keyboard shortcuts (⌘+K for command palette, ⌘+Enter to generate)
- Infinite scroll for history table (or pagination)
- Responsive sidebar (collapsible on mobile, hamburger menu)

## 🏁 Implementation Order (Recommended)

1. **Extract components & restructure folders** (low risk, high maintainability)
2. **Add React Router + design system tokens** (unblocks all UI work)
3. **Dashboard page** (quick win, shows value)
4. **Inline editing + enhanced table** (core UX improvement)
5. **Dark mode** (high user satisfaction)
6. **New export formats** (quick features)
7. **Script generation enhancements** (API, BDD, performance)
8. **Backend migration to Express + validation** (stability)
9. **Testing & CI/CD** (quality gate)
10. **Collaboration features** (team value)

## ⚠️ Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Large refactor breaks existing flows | Incremental extraction with feature flags; keep old paths working temporarily |
| State migration introduces bugs | Add comprehensive integration tests before migrating |
| Backend migration causes downtime | Run Express on different port, proxy via Vite middleware, flip traffic |
| Scope creep | Lock scope per phase; deliver MVP per phase then iterate |
